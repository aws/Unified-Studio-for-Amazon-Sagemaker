# Account Pool Factory — Web UI

Two serverless UIs: **Project Creator** and **Pool Management Console**.

## Run locally (mock mode)

```bash
cd experimental/AccountPoolFactory/ui
python3 mock-server.py
```

Then open:
- http://localhost:8080/project-creator/
- http://localhost:8080/pool-console/

No npm, no install, no build step. Requires Python 3.6+.

## Structure

```
ui/
  mock-server.py          # local dev server (static files + mock API)
  mock/
    api.py                # mock API route handlers
    data.py               # sample data (accounts, profiles, users)
  project-creator/
    index.html
    app.js
    style.css
  pool-console/
    index.html
    app.js
    style.css
```

`config.js` and `auth.js` are served dynamically by the mock server:
- In mock mode: `config.js` points `API_URL` at `localhost:8080/api`, auth is a no-op
- In production: these files are generated by the deploy script from CF stack outputs

## Switch to real backend

Run the deploy script (once infrastructure is ready):

```bash
./scripts/02-domain-account/deploy/04-deploy-ui.sh
```

This generates `config.js` from CF outputs and uploads everything to S3.
