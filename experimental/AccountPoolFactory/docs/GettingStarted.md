## Documentation Philosophy

We've organized documentation into three levels:

1. **User Guide** - What you need to know to use the system
2. **Development Progress** - What we learned building it
3. **Testing Guide** - How to test every detail

This keeps user-facing docs clean while preserving the development journey for contributors.
